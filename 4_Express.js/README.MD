## âš¡ **What is Middleware?**

Middleware is a **function that sits between the clientâ€™s request and the serverâ€™s response**.

It can **modify requests, log data, check authentication, or handle errors** before the request reaches the final route.

ðŸ§  **Think of it as:**

> Security guard ðŸ›¡ï¸ checking requests before they enter your app.

### âœ… **Example:**

```jsx

const app = express();

// First middleware
app.use((req, res, next) => {
  console.log("Middleware 1 Running");
  next(); // passes control to the next middleware or route
});

// Second middleware
app.use((req, res, next) => {
  console.log("Middleware 2 Running");
  next();
});

app.get("/", (req, res) => {
  res.send("Hello World!");
});

app.listen(3000);
```

### ðŸ§© **Key Points:**

- `app.use()` â€“ used to apply middleware.
- `next()` â€“ moves to the next middleware or route.
- Middleware runs **in order** they are defined.
- You can use middleware for:
  - Logging
  - Authentication
  - Parsing data (`express.json()`, `express.urlencoded()`)

## ðŸ§¯ **Error Handling in Express**

Express provides a special middleware for handling errors.

### âš ï¸ Example:

```jsx

app.get("/", (req, res) => {
  res.send("Hello World!");
});

  return next(new Error("Not Implemented"));
});

// Error handling middleware
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).send("Something broke!");
});

app.listen(3000, () => console.log("Server running on port 3000"));
```

### ðŸ§© **How It Works:**

- `next(new Error(...))` â€“ passes an error to Express.
- Express automatically skips all other middleware and runs the **error handler**.
- `err.stack` â€“ gives the error trace (helpful for debugging).
- Always define error middleware **at the bottom** of all routes.
